# Project Name
LunaVoice

# One-Sentence Summary
A voice-first wellness journal that helps people track their health and cycle patterns by simply speaking about how they feel.

# Why This Experiment Exists
We wanted to see if a voice-first interface could make health tracking feel less like a chore and more like a supportive conversation. Our goal was to explore whether we could extract useful wellness signals (like pain levels or energy) from natural speech and present them back to the user in a way that helps them prepare for doctor visits without using clinical or diagnostic language.

# What We Tried
- **Minimal "Speak" Interface:** A home screen with one central button to encourage immediate sharing.
- **Automated Signal Extraction:** A system that transcribes speech and identifies structured data like pain types, energy levels, and sleep quality.
- **Soft Emotional Context:** Identifying gentle emotional cues (e.g., "overwhelmed" or "steady") to provide context for the user's entries.
- **User-in-the-Loop Review:** A screen that allows users to confirm, edit, or delete the extracted information before it is saved.
- **Doctor-Ready Summaries:** A simple text-based summary of 30 days of data, designed to be shared during a medical appointment.

# What We Learned
- **Simplicity over Dashboards:** For wellness and self-advocacy, a minimal layout felt safer and more private than a data-heavy dashboard.
- **The Importance of Editing:** Users need to feel in control of the "AI's interpretation." The review step is essential for building trust.
- **Language Matters:** Framing emotional cues as "possible cues" rather than "results" helped keep the experience supportive rather than clinical.
- **Simulated vs. Real Voice:** While the current prototype uses simulated processing, the interface design proved that a voice-first flow is technically viable for this use case.

# What We’d Try Next
- **Real-Time Processing:** Moving from simulated transcription to real-time voice-to-text.
- **Clinician Feedback:** Showing the "doctor-ready summary" to actual medical professionals to see if it provides the right level of detail for an appointment.
- **Cycle Data Integration:** Connecting to existing cycle tracking APIs to replace manual day-counting with real data.
